Step 2

Photo to Feature Map to AR Overlay

Before any 3D output, the Taxila photo was run through real feature detection, the same math a reconstruction pipeline actually uses to find matchable points across images, then an AR-style line overlay simulating what a heritage app would show live on a phone camera.

Stage 1

ML Feature Detection

Sobel gradients and 24,091 Harris corner points, concentrated on the monastery's cell walls, exactly what Structure from Motion matches across photos.

Stage 2

AR Line Overlay

25 tightened Hough line segments tracing real structural edges. An earlier pass produced 271 noisy lines before the detection thresholds were fixed.

Capture-to-3D Preview

What this is, honestly: a monocular heuristic depth estimate, not a trained network like MiDaS and not multi-view triangulation. Real multi-view Structure from Motion (ORB feature matching plus RANSAC) was tested first on a curated multi-angle photo set from another module and found only 12–17 geometrically consistent matches out of 2,000+ keypoints per photo pair, not enough overlap for real triangulation from photos taken by different photographers rather than a controlled capture. This point cloud is the honest alternative for a single Taxila photo: a monocular depth heuristic, sky and haze masked out by brightness, exported as a real, hand-validated PLY file.
